## 并行层级说明

| 层级 | 位置 | 并行方式 | 状态 | 效果 |
|------|------|----------|------|------|
| 进程级 | run_parallel.py | 多进程 (每个进程处理多个视频) | ✅ 正常 | **最佳** - 充分利用多核 |
| 视频级 | main_pipeline.py | 多线程/进程 (--workers参数) | ❌ 失效 | num_workers=1硬编码 |
| Round级 | main_pipeline.py | 多进程 (pose检测) | ✅ 正常 | 一般 |
| 样本提取级 | main_pipeline.py | 多进程 (ProcessPoolExecutor) | ✅ 正常(已禁用) | 收益小 |
| 特征计算级 | main_pipeline.py | 多进程 (ProcessPoolExecutor) | ✅ 正常(已禁用) | 收益小 |

## 使用建议

### 推荐：使用 run_parallel.py

```bash
# 处理1000个视频，使用8个worker
python run_parallel.py --workers 8 --max-videos 1000 --timeout 300
```

### 不推荐：直接使用 main_pipeline.py

```bash
# --workers 参数无效，实际是单进程
python main_pipeline.py --batch --max-videos 100 --workers 8
# 效果等同于 --workers 1
```

## 进程隔离原理

run_parallel.py 使用 **subprocess.Popen** 启动独立进程：

```python
# 每个Worker是独立的Python进程
process = subprocess.Popen(
    cmd,                    # python main_pipeline.py --batch ...
    stdout=subprocess.PIPE, # 输出重定向到文件
    stderr=subprocess.STDOUT,
    cwd=work_dir,
    env={...}               # 环境变量独立
)
```

优势：
- 每个Worker有独立的内存空间
- 不会相互影响
- 一个Worker崩溃不会导致其他Worker崩溃

## 内部并行禁用原因

当使用 run_parallel.py 时，main_pipeline.py 内部并行会被禁用：

- 环境变量 `DISABLE_INTERNAL_PARALLEL=1`
- 避免嵌套并行导致的进程爆炸
- 防止多核CPU被耗尽
