# 速度加速度验证团队 - AGENTS规范

## 概述

本规范定义了一个用于验证数据合理性的团队协作模式，适用于分析和验证速度和加速度等物理特征的正确性。

---

## 团队配置

| 角色 | 职责 | 交付物 |
|------|------|--------|
| **Worker-1** | 数据结构分析 | CSV结构分析报告、列分类 |
| **Worker-2** | 速度数据验证 | 速度合理性评估、异常值检测 |
| **Worker-3** | 加速度数据验证 | 加速度合理性评估、异常尖峰检测 |

---

## 工作流程

### 阶段1: 团队创建

1. **TeamCreate**: 创建专用团队名称（如 `velocity-accel-verification`）
2. **TaskCreate**: 创建3个任务，分别对应3个验证维度
3. **TaskUpdate**: 预分配owner避免竞争

### 阶段2: 并行执行

- 3个worker并行工作，各自独立分析分配的任务
- 不互相依赖，通过消息汇报给team-lead

### 阶段3: 结果汇总

- Team-lead收集各worker的分析结果
- 生成综合验证报告

### 阶段4: 清理

- 发送shutdown_request到所有worker
- TeamDelete清理团队资源

## 任务定义模板

### 任务#1: 结构分析

```
分析{csv_path}文件结构。

1. 读取CSV前20行
2. 对所有feat_*列计算统计信息（均值、标准差、最小最大值）
3. 识别哪些列是：位置坐标(x,y)、速度(vx,vy)、加速度(ax,ay)
4. 输出结构化分析报告到csv_structure_analysis.md
```

### 任务#2: 速度验证

```
验证{csv_path}中速度数据的合理性。

速度合理性标准：
1. 数值范围：人体运动速度通常在0-500像素/秒
2. 方向变化：速度方向变化应该平滑
3. 物理一致性：Y方向速度通常为负值（下落）

分析内容：
1. 提取速度相关列（vx, vy）
2. 计算统计信息：均值、标准差、最小最大值、分位数
3. 检查是否存在异常值（过大或过小）
4. 输出验证结论
```

### 任务#3: 加速度验证

```
验证{csv_path}中加速度数据的合理性。

加速度合理性标准：
1. 数值范围：人体运动加速度通常在-5000到5000像素/秒^2
2. 变化平滑：加速度变化应该平滑，无高频噪声
3. 物理定律：加速度应该与速度变化一致

分析内容：
1. 提取加速度相关列（ax, ay）
2. 计算统计信息：均值、标准差、最小最大值、分位数
3. 检查是否存在异常尖峰（3σ外）
4. 验证配对列是否互为相反数
5. 输出验证结论
```

---

## 验证标准参考

| 物理量 | 合理范围 | 异常阈值 |
|--------|----------|----------|
| X方向速度 | -500 ~ 500 像素/秒 | >600 或 <-600 |
| Y方向速度 | -500 ~ 500 像素/秒 | >600 或 <-600 |
| X方向加速度 | -3000 ~ 3000 像素/秒² | >5000 或 <-5000 |
| Y方向加速度 | -3000 ~ 3000 像素/秒² | >5000 或 <-5000 |

### 人体运动参考值

- 慢走: ~100 像素/秒
- 快走: ~200 像素/秒
- 跑步: ~300-500 像素/秒
- 重力加速度: ~9810 像素/秒² (假设30像素/厘米)

## 使用示例

```
/team 3:executor "验证{csv_path}中速度和加速度数据的合理性"
```

执行流程：
1. Lead解析参数，创建团队 `velocity-accel-verification`
2. 创建3个任务并预分配owner
3. 启动3个executor worker并行分析
4. 收集结果，生成验证报告
5. 关闭团队，清理资源
